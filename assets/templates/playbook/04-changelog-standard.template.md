# Changelog Standard

## Format

- Version header: `## [X.Y.Z] - YYYY-MM-DD`
- Sections: `Added`, `Changed`, `Fixed`, `Removed`, `Security` (as needed)

## Rules

- Describe user-visible behavior impact
- Keep entries concise and verifiable
- Update changelog in the same change as version bump
- Skill-first: `{{SKILLS_ROOT}}/draft-changelog-entry.md` can draft entries first.
