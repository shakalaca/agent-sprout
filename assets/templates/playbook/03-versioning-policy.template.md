# Versioning Policy

Use Semantic Versioning: `MAJOR.MINOR.PATCH`.

- PATCH: Fixes and non-contract adjustments
- MINOR: Backward-compatible behavior or API additions/changes
- MAJOR: Breaking changes

- Skill-first: `{{SKILLS_ROOT}}/decide-version-bump.md` can draft the decision first.

If uncertain, choose the higher impact level and document why.
